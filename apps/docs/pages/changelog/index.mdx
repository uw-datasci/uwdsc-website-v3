# Timeline

Track the evolution of the UWDSC websites and applications. Features are organized by semester to align with the academic calendar.

## How This Works

Features are automatically documented from merged Pull Requests that are tagged with semester labels following this naming convention:

### Semester Label Format

Semester labels follow the pattern: `<Season><Year>`

- **F** = Fall
- **W** = Winter
- **S** = Spring
- **YY** = Two-digit year

### Documentation Approach

Each semester page focuses on **what** was built and **why** it matters to users, rather than technical implementation details. Features are automatically added to the appropriate semester page when PRs with matching labels are merged.
