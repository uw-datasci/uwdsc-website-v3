# Getting Started

Get your development environment up and running in minutes.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** >= 24 (see `engines` in root `package.json`)
- **pnpm** 11.x (pinned via `packageManager` in root `package.json`; install through Corepack below)

### Installing pnpm (Corepack)

Use [Corepack](https://nodejs.org/api/corepack.html), which ships with Node.js, to enable the `pnpm` shim on your PATH. This matches the approach described in the Node.js documentation for managing package managers.

```bash
corepack enable
```

On Windows, run the same command in PowerShell or Command Prompt; you may need an elevated shell the first time.

After Corepack is enabled, running `pnpm` inside this repository uses the exact version declared in the root `package.json` `packageManager` field (currently pnpm 11.x) without a separate global install.

To install pnpm 11 globally (for example, before you clone the repo), run:

```bash
corepack install -g pnpm@11
```

Or follow the [official pnpm installation guide](https://pnpm.io/installation).

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/uw-datasci/uwdsc-website-v3.git
cd uwdsc-website-v3
```

### 2. Install Dependencies

```bash
pnpm install
```

This will install all dependencies for all packages and apps in the monorepo. Thanks to pnpm workspaces, shared dependencies are efficiently linked.

### 3. Set Up Environment Variables

First, create a `.env` file in the **root directory** of the repository:

```bash
INFISICAL_CLIENT_ID=<insert-id-here>
INFISICAL_CLIENT_SECRET=<insert-secret-here>
```

> **Note**: Contact the Dev VPs on Discord to get your Infisical credentials, or check the DSC exec server for the values.

Then, pull environment variables from Infisical:

```bash
pnpm pull-secrets
```

The `pull-secrets` script will:

- Install the Infisical CLI globally if it's not already installed
- Initialize Infisical configuration if needed
- Authenticate with Infisical (using machine identity or user login)
- Sync environment variables from Infisical for `web` and `admin` apps
- Create `.env.local` in each app directory with the synced secrets

## Running the Apps

### Start All Apps

```bash
pnpm dev
```

This starts all applications in development mode:

- **Web app**: http://localhost:3000
- **Admin app**: http://localhost:3001
- **Docs**: http://localhost:3002

### Start Specific Apps

If you only want to work on a specific app:

```bash
# Main website only
pnpm dev:web

# Admin app only
pnpm dev:admin

# Documentation only
pnpm dev:docs
```

## Building for Production

### Build All Packages

```bash
pnpm build
```

### Build Specific App

```bash
pnpm build --filter=web
pnpm build --filter=admin
pnpm build --filter=docs
```

# Development

## Troubleshooting

### Port Already in Use

If you get an error that a port is already in use, you can either:

1. Kill the process using that port
2. Change the port in the app's `package.json` dev script

### pnpm Installation Issues

If you encounter issues with pnpm, try:

```bash
# Clear pnpm cache
pnpm store prune

# Reinstall dependencies
rm -rf node_modules
rm pnpm-lock.yaml
pnpm install
```

### TypeScript Errors

If you see TypeScript errors, try:

```bash
# Clean and rebuild
pnpm clean
pnpm build
```
